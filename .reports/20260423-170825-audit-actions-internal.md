## Duplicates — Internal

Rare but possible — two private scripts inside one action that could be merged without changing the action's contract.

- The `git ls-remote --tags` + URL-token assembly block is duplicated across `bump-patch-versions`, `check-tag-pattern-exists`, `publish-tag`, `create-component-tags`, `resolve-commit`, `resolve-latest-prerelease-tag`, `resolve-latest-tag-from-sha`, `validate-tag-exists` — 8 actions. All 8 emit the same 5 lines of `if TOKEN; then remote_url=...@HOST/REPO.git; else remote_url=https://HOST/REPO.git; fi`. Candidate for a shared bash helper in `shared/` (the repo already has `shared/gh-retry.sh` and `shared/_lint/check-no-pwsh.sh` per the rubric, so the precedent exists).
  - **Recommended (best-practice):** add `shared/remote-url.sh` that exports a function `remote_url_for TOKEN HOST REPO` echoing the composed URL, and source it from each of the 8 actions. Deletes ~5 lines per action × 8 actions = ~40 lines of near-duplicated code.
  - Internal-scope (helper addition; no consumer surface change). Classified non-breaking by definition of internal.
  - **Proposed for the plan.**

- The `http.<host>.extraheader` / `includeIf.gitdir:` clearing block is duplicated across `create-component-tags` and `publish-tag` (the two actions that `git push` under a URL-token). Both have the same ~10-line bash that unsets both forms of actions/checkout credential storage. Candidate for the same shared helper (or a sibling one).
  - **Recommended (best-practice):** bundle with the above as `shared/git-push-helpers.sh`. No consumer surface change.
  - Internal-scope, non-breaking.
  - **Proposed for the plan** (combined item with the above).

- `gh-retry.sh` is already shared and sourced from `check-commit-status-exists`, `cleanup-github-prereleases` (via external script), `commit-files`, `create-commit-status`, `get-commit-status`, `get-last-successful-github-workflow-run-timestamp`, `trigger-and-wait-for-github-workflow`, `wait-for-github-workflow`. Good use of the pattern already.
