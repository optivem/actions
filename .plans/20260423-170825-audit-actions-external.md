## Follow-up review

- [ ] **Audit every `validate-*` action for a corresponding `check-*` predicate** — systematically review whether each assertion-style `validate-*` should be split into a pure `check-*` primitive (boolean output, never fails) + a thin `validate-*` wrapper (assertion that fails on violation), following the convention this repo already established for `check-tag-exists` / `validate-tag-exists`.

  **Mainstream best-practice guidance:**
  - There is no universal GitHub Actions / CI ecosystem convention mandating this split (e.g. `actions/cache` uses `save` / `restore` rather than `check` / `validate`, and most linters use flags like `--fix` vs read-only rather than separate tools).
  - However, the `check-*` (predicate) / `validate-*` (assertion) split IS the convention this repo has chosen, and it is an **application of the broader "query/command separation" principle** (Meyer's CQS): separate the question-asking shape from the action-taking shape so callers can consume either without side effects leaking.
  - Rubric §5 (primitives first, composites optional and thin) supports this split **when a real caller needs the predicate shape**, but warns against pre-emptive splits for actions where every caller wants the assertion and nothing more. **YAGNI governs.**

  **Criteria for pulling the predicate out:**
  1. At least one caller wants the soft-predicate shape (evaluate without failing) AND at least one wants the assertion (fail on violation). A single shape across all callers means the split adds surface area for no gain.
  2. The validate action currently bundles a skip-mode flag or optional output that exposes the check's outcome — this is a tell that callers want the predicate but the current shape forces them through the validate composite.
  3. The check logic is a meaningful re-usable primitive (not "is this single string non-empty" or similar trivial predicate).

  **Per-action recommendations** (audit snapshot; re-evaluate as call-site patterns evolve):
  - `validate-tag-exists` — HAS a primitive (`check-tag-exists`). ✅ Done.
  - `validate-env-vars-defined` — 58 callers, all hard-fail on missing. No known predicate use case (callers don't ask "are env vars set?" without wanting to fail). **Defer** — add `check-env-vars-defined` only when a concrete predicate-shape caller emerges.
