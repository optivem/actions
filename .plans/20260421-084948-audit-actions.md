- [ ] **Split `generate-prerelease-version` into a wrapper that does not duplicate `compose-prerelease-version`** ÔÇö `generate-prerelease-version` is a thin composite that (a) calls `compose-prerelease-version` and (b) asserts the final release version is unreleased. Rubric ┬ž5 says thin composites are fine only when the composite adds no hidden logic; this one hardcodes `number: ${{ github.run_number }}` where `compose-prerelease-version` takes an arbitrary `number`. That coupling is the only value-add, but it silently ties the RC number to `run_number`. Options: (a) move the guard (`ensure-version-unreleased`) into the caller workflows and delete `generate-prerelease-version` entirely; (b) rename to `generate-rc-version-from-run-number` to make the binding explicit; (c) expose `number` as an input with `default: ${{ github.run_number }}`. Recommended (c): preserves the run-number default, but stops making it magical.
