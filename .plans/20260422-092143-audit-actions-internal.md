## DevOps alignment findings — Internal

### Tool-agnostic composition

- **`bump-patch-versions` hardcodes `origin` as the remote.** Every other Tier 2 action in this repo accepts a `git-host` input (`validate-tag-exists`, `publish-tag`, `check-tag-pattern-exists`, `create-component-tags`, `resolve-commit`, `resolve-latest-prerelease-tag`, `resolve-latest-tag-from-sha`). The inconsistency means a student porting to GitLab/self-hosted would have to edit *this* action while leaving its siblings alone. Rubric §3.1 Tier 2 sub-rule. Recommended: add `git-host` input with default `github.com` and compose the remote URL the same way siblings do — additive with behaviour-preserving default, no consumer impact.

The rubric exempts missing `branding:` as a finding under internal-only publication, and the spec explicitly says do NOT flag missing `branding:`. But the *inconsistency* is still a code-quality smell — pick one shape and apply it uniformly.
